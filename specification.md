# Fanfic Prompt Maker - Specification

## 1. Overview

A lightweight, browser-based tool that acts as an **interactive prompt engineer for fanfiction**. It does not write the fic itself — instead, it interviews the user, understands their creative intent, and generates a richly-formatted master prompt ready to be pasted into ChatGPT, Gemini, or Claude.

No build tools. No framework. No installation. Just open `index.html` and go.

---

## 2. Technology Stack

| Layer | Technology |
|---|---|
| Structure | HTML5 |
| Styling | Vanilla CSS (with CSS custom properties for theming) |
| Logic | Vanilla JavaScript (ES Modules) |
| AI Backend | Google Gemini API (`@google/genai` via ESM.sh CDN) |
| Fonts | Google Fonts (e.g. Merriweather for headings, Inter for body) |

No npm, no build step, no dependencies to install. The app loads directly in a browser.

## 4. User Flow (Step-by-Step)

### Step 0 — API Key Screen
- The very first screen shown on load.
- A clean, centered card with:
  - A short description of the tool.
  - A password-type input field: `Enter your Google Gemini API Key`.
  - A "Start" / "Let's Go" button.
- The key is stored in `sessionStorage` (cleared when browser tab is closed).
- On confirmation, the app transitions to Step 1.

---

### Step 1 — Context & Intent
- **Optional File Upload**: A drag-and-drop / click-to-upload zone accepting `.pdf`, `.docx`, and `.md` files. The file is read client-side and its text content is extracted.
- **Intent Input**: A large, multi-line `<textarea>` prompting the user with placeholder text like: *"Describe the fanfic you want to write. Who are the characters? What's the scenario? Any specific tone or events?"*
- A **"Generate Questions"** button at the bottom.
- On submit, app shows a loading state and calls the Gemini API.

---

### Step 2 — The 5 Questions (Interactive Q&A)
- The Gemini API is called with the user's intent (+ extracted context if provided) to generate **5 targeted, relevant questions** with **3 suggested answer options** each.
- Questions are displayed **one at a time**, card by card, to keep focus.
- Each question card contains:
  - The question text.
  - **Option A, B, C**: Radio-button selectable answers generated by the AI.
  - **Option D — "Write my own"**: Selecting this reveals a text input for a custom free-text answer.
  - A **"Confirm Answer"** button to lock in the choice and advance to the next question.
- A visual progress indicator (e.g. "Question 2 of 5") is shown throughout.

---

### Step 3 — Prompt Generation
- After all 5 answers are confirmed, the app shows a loading / "thinking" state.
- The Gemini API is called a final time to synthesize all inputs into the final master prompt.
- The generated prompt is displayed in a styled, read-only text area.
- A **"Copy to Clipboard"** button is prominently placed.
- A **"Start Over"** button resets to Step 1 (keeping the API key in session).

---

## 5. Final Prompt Format (Directives)

The final prompt is **not a summary** — it is a fully-formed, detailed creative brief designed to make the destination AI write exactly what the user envisioned. The prompt must be:

- **Long and detailed**: It should fill in every relevant detail rather than leaving gaps. Vague prompts produce vague fic. The AI should be given no room to guess.
- **In-character and accurate**: Characters must be described with proper voice, personality, and behavioral quirks as established in the source material or as specified by the user. The destination AI must be instructed to stay true to how each character *actually speaks and thinks*, not a generic approximation.
- **Full-length conversation ready**: The prompt should set up a full scene or chapter, not just a scenario sketch. It should include context on where the scene starts, how it escalates or progresses, and where it ends (or what the emotional beat of the ending should be).

---

### 5.1 Writing Style Directives

The following directives are embedded into the generated prompt. Each one can be toggled on/off by the user via the **Settings Panel** (see §6):

| # | Directive | What it instructs the destination AI to do |
|---|---|---|
| 1 | **Onomatopoeic Dialogue** | Weave sound words naturally into narration and speech (e.g. *thud*, *hiss*, *click*). |
| 2 | **Vivid Sensory Description** | Describe every scene with sensory detail — what is seen, heard, felt, smelled. Show, don't tell. |
| 3 | **Inward Thinking / Internal Monologue** | Include the POV character's unfiltered inner voice in italics, revealing doubts, feelings, and reactions. |
| 4 | **Paragraph Variety** | Mix short, punchy beats with longer flowing paragraphs to control pacing and emotional rhythm. |
| 5 | **Character Voice Accuracy** | Each character must speak and react in a way that is consistent with their established personality. Include specific notes about the character's speech pattern, quirks, or typical behavior pulled from the user's input and Q&A answers. |
| 6 | **Dialogue-Heavy Scene** | The scene should be primarily driven by conversation between characters, with action and description woven in between exchanges. |
| 7 | **Emotional Depth** | The narrative must make the reader *feel* the moment — use dramatic tension, vulnerability, or warmth depending on the scene's tone. |
| 8 | **Scene Continuity Cue** | At the end of the generated chapter/scene, the AI should naturally lead into the next beat rather than ending abruptly. |

---

### 5.2 Appended Confirmation String

The very end of every generated prompt must have the following string **manually appended** (hardcoded in `app.js`, not AI-generated, regardless of settings):

```
---
Before you begin writing, please read everything above carefully. Confirm that you understand the characters, their voices, the scenario, and the intended tone. If anything is unclear or you have questions, ask now. Once you are ready, let me know and we will begin.
```

This ensures the destination AI always pauses to check in before generating content, reducing out-of-character or off-tone results.

## 6. Settings Panel

A collapsible or slide-in **Settings Panel** accessible at any point during the session (not just at the end). It allows the user to fine-tune exactly what instructions get baked into the final generated prompt.

### 6.1 Writing Style Toggles
A set of checkboxes, all **checked by default**, corresponding to the directives in §5.1. The user can uncheck any directive they do not want included.

```
[✓] Onomatopoeic Dialogue
[✓] Vivid Sensory Description
[✓] Inward Thinking / Internal Monologue
[✓] Paragraph Variety
[✓] Character Voice Accuracy
[✓] Dialogue-Heavy Scene
[✓] Emotional Depth
[✓] Scene Continuity Cue
```

### 6.2 Prompt Length Preference
A radio group that adjusts how long and thorough the final generated prompt is:

```
( ) Concise   — Short, tight prompt. Fast to paste and use.
(●) Standard  — Balanced detail. Recommended.
( ) Detailed  — Maximum context. Best for complex multi-character scenes.
```

### 6.3 POV Mode
A radio group controlling what point of view the destination AI should write in:

```
( ) First Person  (I saw, I thought...)
(●) Third Person Limited  (She saw, he thought... — follows one POV character)
( ) Third Person Omniscient  (narrator sees all)
```

### 6.4 Tone Hints (optional free-text)
A small text input where the user can add any additional stylistic note that doesn't fit anywhere else (e.g. *"Keep it hopeful"*, *"Dark and melancholic"*, *"Slow burn, build tension gradually"*). This string is appended verbatim into the generated prompt's tone section.

### 6.5 Settings Persistence
Settings are saved to `localStorage` so they persist between sessions. A **"Reset to Defaults"** button is included at the bottom of the settings panel.

## 6. Design & Aesthetic

The visual style should feel like a **premium, dark-mode writing tool** — inspired by AO3's literary feel but elevated with a modern, sleek execution.

- **Color palette**: Deep charcoal/dark navy background, warm cream or soft gold accent, muted slate for secondary text.
- **Typography**: `Merriweather` (serif) for question/prompt text. `Inter` (sans-serif) for UI chrome.
- **Transitions**: Smooth fade/slide transitions between each step/card.
- **Card UI**: Questions and steps presented as floating cards with subtle shadow and border.
- **Responsive**: Should be usable on a laptop and a large tablet.

## 7. Gemini API Integration

### 7.1 Generating the 5 Questions
- **Model**: `gemini-2.0-flash` (fast, low-latency for dialogue)
- **Input**: User intent text, optionally prepended with extracted document context.
- **System Prompt**: Instructs the model to return a structured JSON array of 5 question objects, each with a `question` string and a `options` array of 3 strings.
- **Output Parsing**: The response is parsed from JSON and stored in the app state.

### 7.2 Generating the Final Prompt
- **Model**: `gemini-2.0-flash`
- **Input**: The original intent, context (if any), and an ordered list of all 5 Q&A pairs.
- **System Prompt**: Instructs the model to write a detailed, creative-fiction prompt incorporating all the answers, styled for the target format described in §5.
- **Output**: Plain text. The hardcoded confirmation string (§5) is appended by `app.js` after receiving the response.

---

## 8. State Machine

The app progresses linearly through the following states managed in `app.js`:

```
API_KEY_ENTRY → INTENT_INPUT → GENERATING_QUESTIONS → QA_SESSION → GENERATING_PROMPT → PROMPT_DISPLAY
```

Each state corresponds to a different "screen" shown/hidden via CSS class toggling on a root container. No routing library needed.

---

## 9. Out of Scope (v1)

- Account system or saved sessions.
- Multiple prompt modes or genre-specific templates (may be added in v2).
- Direct posting to AO3 or any other platform.
- Editing/regenerating individual questions after generation.
